# BGIguard 需求规格文档

## 1. 项目概述

**项目名称**: BGIguard (BetterGI 守护程序)

**项目类型**: Windows 桌面工具 / 游戏辅助启动器

**核心功能**: 实现 BetterGI 启动器的"无痛"替换，提供崩溃自启、掉线重连、单实例保护、游戏进程联动等守护功能。

**目标用户**: 使用 BetterGI 启动器的《原神》玩家，需要稳定挂机辅助的玩家群体。

---

## 2. 功能需求

### 2.1 核心功能列表

| 功能 | 描述 |
|------|------|
| 崩溃自启 | 监控 BetterGI/BGI 进程，异常退出时自动重启 |
| 掉线重连 | 检测游戏进程状态，断线后自动重连 |
| 单实例保护 | 防止多开，确保只有一个守护实例运行 |
| 游戏进程联动 | 监控原神客户端进程状态 |
| 设置界面 | 无参数启动时弹出设置窗口，可配置参数 |
| 日志记录 | 实时写入日志文件，按日自动轮转 |
| 内存检测 | 监控 BGI.exe 内存使用，超阈值自动重启 |

### 2.2 文件伪装与替换机制

**初始状态**:
- 用户提供两个文件: `BetterGI.exe`(原始启动器) 和 `BGIguard.exe`(守护程序)

**部署机制**:
1. 首次运行守护程序时:
   - 检测自身进程名是否为 `BetterGI.exe`
   - 若不是，则执行文件替换操作:
     - 判断 `BetterGI.exe` 文件大小:
       - **大于 50MB**: 重命名为 `BGI.exe`
       - **小于等于 50MB**: 备份为 `BetterGI.exe.bak`
     - 将自身复制并重命名为 `BetterGI.exe`
     - 保留原始守护器文件 (`BGIguard.exe`) 以便下次更新
   - 以 `BetterGI.exe` 身份重新启动并继续执行

2. 此后用户点击 `BetterGI.exe` 时:
   - 实际启动的是守护程序
   - 守护程序解析启动参数
   - 将参数传递给真正的 `BGI.exe` 执行
   - 若原文件被备份为 `BetterGI.exe.bak`，则传递给 `BetterGI.exe.bak` 执行

3. **后续更新**: 更新 BetterGI 官方版本后，只需再次运行 `BGIguard.exe`，守护器会自动完成新版备份与替换

### 2.3 启动命令传递

守护程序需支持以下命令行参数格式:
```
BetterGI.exe --startGroups 切换队伍 关闭游戏
BetterGI.exe --startOneDragon <配置名称>
```

**处理流程**:
1. 守护程序启动时获取完整命令行参数
2. 将参数完整传递给 `BGI.exe` (原 BetterGI 启动器) 执行
3. 缓存最后一次使用的启动命令，用于重启时调用

### 2.4 进程监控与守护

**监控间隔**: 默认 5 秒 (可调整)

**监控逻辑**:

1. **游戏进程检测**:
   - 检测 `YuanShen.exe` 是否存在
   - 检测 `GenshinImpact.exe` 是否存在
   - 任一存在则视为游戏正在运行

2. **游戏退出处理**:
   - 当 `YuanShen.exe` 和 `GenshinImpact.exe` 均不存在时
   - 自动终止 `BGI.exe` 进程
   - 等待一段时间后自动重启 `BGI.exe` (使用缓存的启动命令)
   - 记录日志

3. **BGI.exe 崩溃处理**:
   - 定期检查 `BGI.exe` 进程是否存活
   - 若 `BGI.exe` 不存在但守护进程在运行
   - 使用缓存的启动命令重启 `BGI.exe`
   - 记录日志

### 2.5 防多开功能

**检测机制**:
1. 程序启动时检查同名进程是否已存在
2. 使用命名互斥体 (Mutex) 确保单实例

**处理逻辑**:
1. 若检测到已存在守护进程:
   - 终止旧守护进程
   - 获取当前启动命令行参数
   - 使用新参数启动 BGI.exe
   - 缓存新启动命令
2. 若不存在:
   - 创建互斥体
   - 正常启动 BGI.exe
   - 缓存启动命令

**说明**: 此设计确保每次双击 `BetterGI.exe` 时，都使用最新的启动参数执行任务，而不是沿用旧的缓存命令。

### 2.6 内存检测功能

**检测对象**: BGI.exe 进程的内存使用情况

**检测方式**: 基于系统整体内存的占用百分比

**默认阈值**:
| 类型 | 默认百分比 | 描述 |
|------|------------|------|
| 内存阈值 | 80% | BGI进程内存占用超过系统总内存的此百分比则重启 |

**配置方式**: 无参数启动时弹出设置窗口，可自定义百分比阈值 (1-100%)

**处理逻辑**:
1. 在监控循环中定期检测 BGI.exe 进程内存使用 (工作集和虚拟内存取较大值)
2. 计算系统总内存，根据百分比计算阈值
3. 若内存超出阈值:
   - 记录警告日志 (包含当前内存使用量和阈值)
   - 终止 BGI.exe 进程
   - 等待一段时间后重启 BGI.exe (使用缓存的启动命令)
   - 记录重启日志

**日志示例**:
```
[2026-04-11 14:40:00.123] [WARN] BGI.exe 内存超限: 12288MB (阈值=12288MB, 80%)
[2026-04-11 14:40:00.456] [INFO] 正在终止并重启 BGI.exe...
```

### 2.7 日志系统

**日志文件**:
- 命名格式: `BGI_guard{日期}.log`
- 例如: `BGI_guard20260411.log`
- 存放位置: 与可执行文件同目录
- **按日追加**，每天生成新的日志文件

**日志内容**:
- 时间戳 (精确到毫秒)
- 日志级别: [INFO] / [WARN] / [ERROR]
- 日志消息

**自动清理**:
- 日志文件数量超过 7 个时自动删除旧日志
- 确保磁盘空间合理使用

**日志示例**:
```
[2026-04-11 14:30:00.123] [BGIguard_v1.0] [INFO] BGIguard 启动成功
[2026-04-11 14:30:00.456] [BGIguard_v1.0] [INFO] 已替换文件 BetterGI.exe -> BGI.exe
[2026-04-11 14:30:05.789] [BGIguard_v1.0] [INFO] BGI.exe 已启动, PID: 12345
[2026-04-11 14:35:00.001] [BGIguard_v1.0] [WARN] 检测到游戏进程退出
[2026-04-11 14:35:00.123] [BGIguard_v1.0] [INFO] 正在重启 BGI.exe...
```

## 5. 配置说明

**设置方式**: 双击 `BGIguard.exe` 无参数启动时，会弹出设置窗口

**可配置项**:
| 配置项 | 默认值 | 范围 | 描述 |
|--------|--------|------|------|
| 内存阈值 | 80% | 1-100% | 系统整体内存占用百分比 |
| 监控间隔 | 5秒 | 1-999 | 守护循环检测间隔 |
| 启动时显示设置 | 开启 | 开/关 | 每次启动是否显示设置窗口 |

**配置文件**: `BGIguard_config.ini` (自动生成)

---

## 6. 注意事项

1. **文件替换不可逆**: 首次运行后，原 `BetterGI.exe` 将被重命名为 `BGI.exe`，此操作不可逆
2. **管理员权限**: 若目录无写权限，可能需要以管理员身份运行
3. **杀毒软件**: 可能被部分杀毒软件误报，请添加信任
4. **游戏兼容性**: 仅支持检测 `YuanShen.exe` 和 `GenshinImpact.exe` 两个进程名

---

## 7. 常见问题

| 现象 | 解决 |
|---|---|
| 双击后闪退 | 检查同目录是否存在 `BGI.exe`；查看 `BGI_guard*.log` 报错 |
| 重复启动多个窗口 | 守护器自带单实例保护，旧进程会被强制结束并重启 |
| 日志暴涨 | 手动删除 `BGI_guard*.log` 即可，后续版本会自动轮转 |
| 关闭游戏后 BGI 还在运行 | **正常现象**：守护器检测到游戏进程消失后会自动终止 BGI，待下次启动游戏时恢复运行。如不希望此行为，请直接退出守护器。 |
